# Memory Policy

## Core Rules

- Capture can be broad; long-term memory must be narrow.
- Source is evidence, not long-term memory.
- Session Summary is continuity, not permanent truth.
- Candidate Memory requires human confirmation before promotion.
- Confirmed Memory must include source, scope, status, and review timing.
- Index files are rebuildable navigation layers, not the knowledge body itself.

## Long-Term Memory Whitelist

Only these information types are eligible for long-term memory:

1. Stable collaboration preferences.
2. Repeated judgment standards.
3. Explicit boundaries or taboos.
4. Core principles of a long-running theme.
5. Verified workflows or methods.
6. Decisions that will continue affecting AI behavior.

## Do Not Directly Store As Long-Term Memory

1. One-off emotions or complaints.
2. Unresolved guesses.
3. Temporary conclusions from a single session.
4. Information that strongly depends on current project state.
5. External opinions unless the user has adopted them as their own judgment.
6. Secrets, credentials, tokens, private keys, or sensitive personal data.

## Promotion Rules

A Candidate Memory should normally satisfy at least two checks before promotion:

- It will repeatedly affect future judgment.
- It holds across multiple contexts.
- The user has expressed it more than once.
- Without remembering it, the user would need to explain it repeatedly.
- It changes how AI should behave in later sessions.

Confirmed Memory requires explicit human confirmation.
