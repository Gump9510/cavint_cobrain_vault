# 00_profile

## What belongs here

Stable profile context, memory policy, collaboration preferences, boundaries, and long-term directions that affect future AI behavior.

## What does not belong here

Raw evidence, project status, ordinary session summaries, or unconfirmed identity/preference claims written as facts.

## When to write here

Write here when background, preferences, boundaries, or long-term direction need durable profile context. Confirmed profile facts require explicit human confirmation; otherwise keep them as candidate/evidence notes.

## Current files

- `memory-policy.md` — memory promotion rules.
- `profile-maintenance.md` — trigger and maintenance workflow for profile notes.
- `user-background.md` — background and experience notes.
- `collaboration-preferences.md` — collaboration preferences and judgment style.
- `long-term-directions.md` — long-term directions and compounding themes.

## Who reads it

AI agents read this before creating or changing memory. Humans read it to understand what the AI is allowed to rely on.

## Common mistakes

- Treating a single session inference as a confirmed profile fact.
- Mixing project state into stable profile context.
- Recording sensitive personal data without explicit instruction.
