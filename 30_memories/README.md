# 30_memories

## What belongs here

Candidate and confirmed long-term memory objects.

## What does not belong here

Ordinary notes, raw chats, session summaries, or unconfirmed preferences written as durable facts.

## When to write here

Create Candidate Memory when a lesson, preference, boundary, principle, workflow, or decision may repeatedly affect future AI behavior. Promote only after explicit human confirmation.

## Who reads it

AI agents read confirmed memories as durable context and candidate memories as proposals requiring care.

## Common mistakes

- Promoting raw chat content directly into Confirmed Memory.
- Creating broad mixed memory blobs.
- Omitting evidence and boundaries.
