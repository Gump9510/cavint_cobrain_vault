# 60_skills-index

## What belongs here

Optional maps and routing notes for AI skills, commands, rules, or workflows that the vault owner actually has installed.

## What does not belong here

Full skill source code, project evidence, long-term memory records, or assumptions that every user has the same external skills installed.

## When to write here

Write here only after the vault owner has installed or created specific reusable skills, commands, or rules and wants to document when to use them.

## Who reads it

AI agents may read it before selecting optional installed skills. Humans read it to understand local workflow extensions. CobrainVault must still work when this directory is empty.

## Common mistakes

- Treating a skill index as the operating protocol.
- Copying large skill implementations into the index.
- Listing skills that are not actually available in the recipient's AI environment.
