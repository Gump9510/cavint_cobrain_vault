# 70_outputs

## What belongs here

Reusable outputs such as articles, scripts, checklists, offers, frameworks, and finished deliverables.

## What does not belong here

Raw source material, session summaries, or durable memory records.

## When to write here

Write here when analysis or project work produces something reusable beyond the session.

## Who reads it

Humans use outputs directly. AI agents read them when continuing writing, packaging, QA, or reuse.

## Common mistakes

- Embedding all evidence inside the output instead of linking context.
- Publishing an output without scope and boundaries.
