# 50_projects

## What belongs here

Concrete initiatives with goals, current state, deliverables, constraints, decisions, risks, and next actions.

## What does not belong here

Raw evidence, durable memory, or broad long-running themes without concrete action.

## When to write here

Create or update a project when work has a goal, state, deliverable, or next action.

## Who reads it

AI agents read projects to continue execution without starting from scratch.

## Common mistakes

- Mentioning a project in session metadata without creating a project file.
- Letting Current State go stale.
- Putting long-term direction into a project when it belongs in a theme.
